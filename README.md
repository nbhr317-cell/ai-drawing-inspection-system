# AI図面審査＆生産管理アシスタント（B2B SaaSプロトタイプ）

製造業の設計・生産管理現場における「図面の目視チェックの手間」と「手戻りコスト」を削減するために開発した、AI組み込み型のフルスタックWebアプリケーションです。

## 💡 開発の背景（ドメイン知識の適用）
前職の製造業（生産管理）の経験から、人間の目視による「表題欄の記入漏れ」や「二重寸法（重複記入）」の見落としが、後工程（加工現場や発注）で大きな手戻り損失を生んでいることに着目しました。
本システムは、熟練者のノウハウをAI（LLM）に学習させ、業務を自動化・仕組み化することを目指して開発しました。

## ✨ 主な機能
1. **図面の自動解析・空間認識（Gemini API連携）**
   - PDFや画像の図面から、表題欄と寸法線を自動認識。
2. **企業固有ルールの動的カスタムエンジン**
   - 画面のチェックボックスに加え、自社の「製図マニュアルPDF」をアップロードするだけで、AIがそのルールを自動抽出し、企業ごとの基準に合わせて図面を審査します。
3. **視覚的エラーハイライト（赤枠オーバーレイ）**
   - AIが返した座標データ（0〜1000の正規化データ）を画像ピクセルに変換し、エラー箇所を正確に赤枠で囲んで設計者に通知。
4. **教育型アドバイス＆模範解答表示**
   - 単なるエラー指摘に留まらず、製図ルールに基づいた「論理的な理由」と「具体的な修正アクション（正面図への集約など）」を提示。ワンクリックで「正しい図面の見本（合格例）」をその場に表示します。
5. **修正指示入り「検収済みPDF」の自動出力**
   - AIの赤枠を焼き付けたPDFをシステムが自動生成。設計者はワンクリックで指示書としてダウンロード可能。

## 🛠️ 技術スタック
- **フロントエンド**: HTML5 / CSS3 / JavaScript (Fetch API, 構造化DOM操作)
- **バックエンド**: Python 3.x / FastAPI (非同期API) / Uvicorn
- **ライブラリ**: PyMuPDF (PDF解析・テキスト抽出) / PIL (画像処理・描画)
- **AI技術**: Google Generative AI SDK (gemini-3.5-flash)

## 🚀 技術的な工夫とデバッグ経験
- **PDF固有の回転・原点バグの解消**: CAD出力PDFの内部にある回転属性による座標ズレを回避するため、一度高解像度画像にラスタライズしてからピクセル単位でImageDraw処理を行い、再PDF化する堅牢なアーキテクチャへリファクタリングしました。
